# Notifications

Notifications keep you up to date on activity across all your games.

## The Notification Bell

The bell icon in the navigation bar shows a count of unread notifications. Click it to see a dropdown of your 20 most recent notifications. Click any notification to go directly to the relevant content.

The dropdown also has a **Mark all read** button (when you have unread notifications) and a delete button on each individual notification.

## The Notifications Page

Click **View all notifications** in the dropdown, or go to **Notifications** in the nav, to open the full notifications page. This shows up to 50 recent notifications with their read/unread status.

**Mark all as read** — If you have unread notifications, a button appears to mark everything as read at once.

**Delete** — Individual notifications can be deleted from either the dropdown or the notifications page.

## What Generates Notifications

| Event | Who receives it |
|-------|----------------|
| New private message | Recipients of the conversation |
| Reply to your comment | You |
| Character @mentioned in a comment | Player(s) whose character was mentioned |
| Action result published | Player who submitted the action |
| GM post in common room | All participants |
| Phase activated | All participants |
| New handout published | All participants |
| Application approved | Applicant |
| Character approved | Player whose character was approved |

## Discord Notifications

You can receive notifications as Discord DMs by connecting your Discord account in [Settings → Notifications](./user-settings#notifications).

Once connected, you can choose which event types trigger a Discord DM. Each type can be toggled on or off independently. Default settings are tuned to send DMs for high-value direct events (messages, action results, approvals, common room posts) and leave high-volume activity (comment replies, character mentions) off by default.

> **Note:** Your Discord account must share a server with the ActionPhase bot for DMs to work. If you receive an error after connecting, make sure the bot has been added to your server.

## Read Tracking

Unread notifications are shown with a highlighted background. Read notifications are dimmed. Clicking a notification marks it as read automatically.

Notification read state is separate from [comment read tracking](./user-settings#reading), which controls how posts and comments in the Common Room are highlighted.
